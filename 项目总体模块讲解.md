# zuolan信号处理FPGA+STM32双核系统 - 总体模块讲解

---
## 项目概述

本项目基于STM32F429IGTx微控制器与EP4CE10F17 FPGA构建的双核信号处理系统，实现了高性能的信号生成、采集、处理与分析功能。系统通过FMC（Flexible Memory Controller）总线实现STM32与FPGA之间的高速双向通信，具备完整的信号链处理能力。

### 系统特点
- **双核架构**: STM32负责算法处理与系统控制，FPGA负责高速信号处理与实时控制
- **高速通信**: FMC总线实现微秒级数据交换
- **完整信号链**: 从信号生成到采集、处理、分析的闭环系统
- **实时性强**: FPGA硬件逻辑与STM32软件算法协同工作

## 硬件平台配置

### 核心处理器
- **STM32F429IGTx**: 主控制器，ARM Cortex-M4内核，最高180MHz
- **EP4CE10F17**: FPGA协处理器，提供并行处理能力
- **系统时钟**: 25MHz晶振，内部PLL倍频

### 关键外设
- **AD9959**: 四通道DDS信号发生器，支持500MSPS输出
- **双路AD9226**: 高速ADC，65MSPS采样率
- **AD9764**: 高速DAC，14位分辨率，125MSPS
- **FMC接口**: 16位数据总线，支持异步读写

## FPGA模块详细说明

### 1. FMC通信控制模块 (FMC_CONTROL.v)

**功能概述**: 实现FPGA与STM32间的高速双向通信接口

**核心特性**:
- 16位并行数据总线
- 支持16个读通道和16个写通道
- 地址锁存与数据分离
- 异步读写操作

**关键信号**:
```verilog
// FMC接口信号
input fpga_nl_nadv;    // 地址锁存使能，低电平有效
input fpga_cs_ne1;     // 片选信号，低电平有效  
input fpga_wr_nwe;     // 写使能信号，低电平有效
input fpga_rd_noe;     // 读使能信号，低电平有效
inout [15:0] fpga_db;  // 双向地址/数据总线
```

**工作机制**:
1. 地址阶段：STM32输出地址，FPGA锁存
2. 数据阶段：根据读写信号进行数据传输
3. 内部路由：根据地址将数据路由到对应功能模块

### 2. DA波形发生器模块 (DA_WAVEFORM_A/B.v)

**功能概述**: 生成高质量的标准波形信号

**支持波形类型**:
- **正弦波**: 1024点查表，14位精度
- **方波**: 占空比可调
- **三角波**: 线性斜坡特性  
- **锯齿波**: 非对称三角波

**技术特性**:
- 10位相位地址输入
- 14位幅度数据输出
- 实时波形切换
- ROM查表方式实现

**控制寄存器**:
```verilog
reg [7:0] WAVEFORM_A;  // 波形选择：0=正弦，1=方波，2=三角，3=锯齿
```

### 3. AD数据处理模块 (AD_DATA_DEAL.v)

**功能概述**: 处理双路ADC数据并提供总线接口

**核心功能**:
- 双路12位ADC数据处理
- FIFO状态监控
- 数据位序转换
- 总线接口适配

**地址映射**:
- `0x0006`: AD1 FIFO数据读取
- `0x0007`: AD1 FIFO状态标志  
- `0x0008`: AD2 FIFO数据读取
- `0x0009`: AD2 FIFO状态标志

### 4. 频率测量模块 (AD_FREQ_MEASURE.v)

**功能概述**: 实现高精度频率测量

**测量原理**:
- 等精度频率测量法
- 门控时间自适应调整
- 32位计数器确保精度

### 5. 时钟与相位控制模块

**MYPLL模块**: 
- 系统时钟生成与分配
- 多相位时钟输出
- 时钟域隔离

**DA_CLK_CTRL模块**:
- DA输出时钟控制
- 相位关系管理
- 同步控制信号

## STM32模块详细说明

### 1. 硬件驱动层

#### AD9959 DDS驱动模块 (AD9959.c/h)

**功能概述**: 四通道直接数字频率合成器控制

**核心特性**:
- 支持四路独立信号输出
- 频率范围：0.1Hz ~ 200MHz
- 相位精度：16位分辨率
- 幅度控制：10位精度

**主要函数接口**:
```c
void AD9959_Init(void);                    // 设备初始化
void AD9959_Set_Fre(u32 Fout);            // 设置输出频率  
void AD9959_Set_Pha(float Pout);          // 设置输出相位
void AD9959_Set_Amp(u16 amp);             // 设置输出幅度
void AD9959_Single_Output(u8 Channel, u32 Fout, float Pout, u16 amp);  // 单通道输出
```

**寄存器配置**:
- 晶振频率：25MHz
- 频率控制字：32位精度
- 相位控制字：16位精度

#### 信号测量驱动 (ad_measure.c, freq_measure.c)

**频率测量特性**:
- 测量范围：1Hz ~ 100MHz
- 测量精度：±0.1Hz
- 自动量程切换

**幅度测量特性**:
- 12位ADC采样
- 多点平均滤波
- 自动增益控制

### 2. 算法处理层

#### FFT频谱分析模块 (my_fft.c/h)

**功能概述**: 基于ARM CMSIS-DSP库的频谱分析

**技术指标**:
- FFT长度：1024点
- 频率分辨率：采样率/1024
- 窗函数：汉宁窗
- 浮点运算精度

**主要功能**:
```c
void fft_init(void);                       // FFT初始化
void calculate_fft_spectrum(float* input_data, uint16_t data_length);  // 计算频谱
float get_precise_peak_frequency(float sampling_freq);  // 获取峰值频率
float calculate_thd(float fundamental_freq, float sampling_freq);  // THD计算
```

**应用场景**:
- 信号频域分析
- 谐波分析
- 频率识别
- 信号质量评估

#### 数字滤波器模块 (my_filter.c/h)

**滤波器类型**:
- **FIR滤波器**: 线性相位特性
- **IIR滤波器**: 高效率实现
- **移动平均**: 简单平滑滤波
- **中值滤波**: 脉冲噪声抑制

#### 卡尔曼滤波模块 (kalman.c/h)

**应用场景**:
- 信号降噪处理
- 状态估计
- 预测滤波
- 动态系统建模

#### 相位测量模块 (phase_measure.c/h)

**测量原理**:
- 互相关算法
- 零交叉检测
- 相位差计算
- 精度：±0.1°

### 3. 应用控制层

#### 任务调度器 (scheduler.c/h)

**调度策略**: 基于时间片的协作式调度

**主要任务**:
```c
// 任务定义
typedef struct {
    void (*task_func)(void);  // 任务函数指针
    uint32_t period;          // 执行周期(ms)
    uint32_t last_run;        // 上次执行时间
} task_t;
```

**预定义任务**:
- `ad_proc`: AD采样处理 (1ms周期)
- `key_proc`: 按键扫描 (10ms周期)
- `comm_proc`: 通信处理 (5ms周期)

#### PID控制器 (app_pid.c/h)

**控制算法**: 增量式PID控制

**参数配置**:
- 比例系数Kp
- 积分系数Ki  
- 微分系数Kd
- 输出限幅

### 4. 通信接口层

#### 串口通信模块 (my_usart.c/h)

**通信参数**:
- 波特率：115200 bps
- 数据位：8位
- 停止位：1位
- 奇偶校验：无

**数据协议**:
- 命令格式：结构化指令
- 应答格式：状态码+数据
- 错误处理：超时重传

#### 数据包处理 (my_usart_pack.c/h)

**封包格式**:
```c
typedef struct {
    uint8_t header;     // 包头标识
    uint8_t cmd;        // 命令码
    uint16_t length;    // 数据长度
    uint8_t* data;      // 数据内容
    uint8_t checksum;   // 校验和
} packet_t;
```

## 技术支持与维护

### 常见问题排查
1. **FMC通信异常**: 检查时序配置和信号完整性
2. **频率测量不准**: 确认晶振频率和时钟分频
3. **FFT结果异常**: 检查采样率和窗函数设置
4. **DA输出失真**: 验证滤波电路和负载阻抗

### 性能优化建议
1. **提高精度**: 使用更高精度的时钟源
2. **降低噪声**: 优化PCB布局和电源设计
3. **增强稳定性**: 添加温度补偿算法
4. **扩展功能**: 集成更多信号处理算法
